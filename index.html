Proven vs. assumed

The same ledger this project pitches to leadership. A tool this capable, presented with full confidence before a backtest exists, sets an expectation the evidence doesn't support yet.

Proven on real data

  • Adapters correctly parse actual Sales, Inventory, and Placement Fee exports — every quirk found by opening real files, not guessed schemas
  • East-region concentration is real and measured directly from your uploaded files, not assumed
  • The LP optimizer is verified against known cost-minimal corner cases and produces a genuinely monotonic cost-vs-coverage curve
  • The manifest adapter auto-detects column headers and flags SKUs with no sales history rather than silently guessing their demand

Still assumed / not yet proven

  • No backtest exists yet. Nobody has compared this tool's recommendation against what was actually shipped and what it actually cost.
  • Without a real Inventory-by-Region file, sell-through correction stays off — raw demand % only
  • The LP optimizer here is a fine grid search over the 3-region simplex, not a general-purpose simplex method — exact for 3 regions, would need real LP for more
  • Data persists in this browser only (localStorage) — not true multi-session disk persistence, and clearing browser data clears it

Real bugs, kept on the record

Not a sanitized changelog — these recurred because they're genuine framework footguns

What this deliberately doesn't do

✕

✕

Under a year of history is too thin to outperform the statistical methods already in use.
